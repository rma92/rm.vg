  1. (20 points) Explain the difference between external, conceputal, and internal schemas. How are these different schema layers related to the concepts of logical and physical data independence?

    The External Schema is a formal definition for user views. The Conceptual Schema integrates the external schema into the Internal Schema, which defines the physical storage structures (e.g. tables, etc.). The External Schema is the most abstract, and logical of the schemas, while the Internal the least abstract, and most closely related to physical modeling.

    The three schemas are all linked, in that in a top-down approach, starting with an External Schema, the Conceptual and Internal Schemas are influenced based on it's design.

  2. (15 points) [Exercise 2.3] What is a data model? Discuss the main types of data model.

    A data model is a description of real-world or other objects represented by a computer system, as well as their properties (attributes) and relationships. There are multiple types of data models, including Database or Record-based Models, Entity-Relationship Models, Semantic Models, Functional Models., and Physical Data Models.

    A Record-based Model, or Database Model specifies a means of storing data in a database. Subtypes include the Relational Data Model, which specifies the overall structure of a database and high-level descriptions of implementation (e.g. data types, links between tables, etc.). Most modern commercial systems are based on this model. These data models do not provide adequate facilities for explicitly specifying the semantics/constraints of the data (e.g. range). Other subtypes include the Network Data Model, which represents data in a graph (data structure), and the Hierarchial Model, which represents data in a tree. The Flat Model, a trivial 2 dimensional array of data, may or may not be included. It may be too trivial for inclusion.

    An Entity-Relationship Model, also called E.R. Model, consists of a diagram of entities with lines, or arrows representing links (the relationships) between entities. For example, in a medical data record system, financial records and medical records may be linked by a patient's social security number.

    A Semantic Model defines the meaning of data within the context of interrelationships with other data. It is an abstraction of data in the system, to the point that it is sometimes called a Conceptual Data Model. The logical data structure of an actual database system cannot satisfy the requirements for semantic specifications of data due to limitations of the types of models implemented in a database. It is therefore necessary to define semantic usage of the data.

    A Functional Model is a structured representation of the functions of a system. This is useful in an object-oriented environment where entities may contain specified functions, in addition to properties. The functions that may be specified included activities, actions, processes, and operations of a particular entity.

    A Physical Data Model defines how data will actually be stored, given constraints, and facilities of an actual database management system. This may include files, or actual information about tables in a database. This is a fairly low level data model.

  3. (15 points) [Exercise 2.5] Describe the types of facility that you would expect to be provided in a multi-user DBMS.

    A multi-user DBMS would be expected to provide facilities that allow for multiple simultaneous users to access the data in a practical, safe, secure, and robust manner.

    Such features would obviously include a Security facility to authenticate users, and prevent inappropriate access of data by other users, or third-parties. This could include simple authentication (a table or file containing usernames and passwords), but it would be preferable to include a more robust system that can be linked to another authentication system such as Active Directory, or a RADIUS server. Access Control Lists would also be extremely desirable, to ensure data integrity.

    Concurrency Control, a Query Language and a View Mechanism of some variety would also be expected, with the first of particular importance. Considering the nature of a multi-user database system, having a facility to maintain the ability of multiple interactions with the database at any given time would be extremely desirable.

    Considering the cost associated with both the database system itself, and the presumed value of the data that will be stored in the database, at least some nature of Crash Recovery or backup system would be expected.

    It would also be expected that the database may be accessed and managed over a network, and to support clustering.
