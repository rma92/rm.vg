  2. (25 points) The owner of a toy store wants to store information (names, address, position, age, etc.) about the employees in the store. Not surprisingly, the volume of data compels the owner to buy a database system. To save money, the owner wants to buy one with the fewest possible features, and plans to run it as a stand-alone application on one machine. Of course, the list will not be shared with other people. Indicate which of the following DBMS features the owner should pay for; in each case, also indicate why the woner should (or should not) pay for the feature in the system.
    1. A security facility;
    2. Concurrency control;
    3. Crash recovery;
    4. A view mechanism;
    5. A query language.

    The majority of these features are likely unnecessary to the scope of the project that the toy store owner is interested in. Since the list will not be shared with other people, and is running on it's own machine, neither a security facility or concurrency control should be necessary. Based on the description, it appears the database will not be online constantly (as in a web application), but will only be brought online as needed. Because of this, Crash recovery should be relatively unnecessary. The desirability of this feature varies indirectly with the frugality of the owner of the toy store. I would recommend not seeking this feature. A view mechanism is very necessary, as without this, it may be troublesome, or at the very least, inconvenient, for the toy store owner to access the database. A query language would also be a desirable feature, as it will aid in management of the database.

  3. (10 points) Using any Web browser, search two different database systems and visit their Web pages. For each of them, please briefly report what you learn from the Web page (e.g. functionalities, characteristics, etc.)

    Oracle Database includes many high end features, such as support for multitenancy, which allows for good cloud consolidation. A large emphasis is placed on data-integrity, as a large amount of protections are provided against server failures, site failures, human errors, and various types of attack ("Defense-in-depth" as they said). It is also extremely scalable.

    MySQL Enterprise Edition includes many high end features as well, but is not as high-end a platform as Oracle's flagship product. The base (free to use) version of MySQL includes basic security and management features. The Enterprise edition adds other database engines, and other types of extensions to enhance data integrity. These include a "hot" backup solution (which allows for quick recovery from a crash), high availability features such as replication, and support for Oracle Solairs Clustering and Windows Failover Clustering. Scalability and Security are enhanced over the free SKU as well, but not to the extent of Oracle. MySQL Thread Pool provides an efficient performance increase, and support for authentication via PAM and Windows Active Directory is available (as well as standard MySQL authentication).

    SQLite is a very different sort of database. It does not provide a server, and is a standalone, small application. The databases are stored in a file. Concurrency is limited to read-only. Only one write operation may occur at a time, as writing locks the database. The scalability of the database is limited due to this, as well as the default size limitations implemented in SQLite. These include a limitation on the size of a database at 4GB, as well as the same cap for runtime memory. It is, however, possible to join multiple large databases that in total take up more than 4GB. SQLite also can create temporary databases in memory, which is a useful feature for certain applications that is not as easy, simple, or performant in MySQL or Oracle, as those databases are quite a bit heavier.
