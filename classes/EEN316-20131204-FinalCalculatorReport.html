Final Project: Calculator

Abstract

The project consists of a calculator with the capability of storing many numbers at once. It can also be provided as an IP core for use in other projects, if desired.

Design

Registers

Registers will be implemented as an array of general purpose memory, R0 through R8. There will be no restrictions on which registers may be used for what purpose, the architecture is very open.

When specifying an OpCode, at least one register will be specified (depending on the instruction). The first register will be where the result is stored. A MOV instruction is also available, which copies between the registers.

OpCode List

Instructions have 4-bit OPCode, either:
  • 3-bit A and 3-bit B inputs.
  • A 3-bit destination and An 8-bit number
InstructionOpCodeDescription
0000 MOV
  • Move contents of B into A.
  • 0000 00 00 is MOV A A, to prevent a glitch
  • To clear a register, use XOR A A.
0001 LD Load into a register from switches. So, 0001 01 would load into register 1
0010 ADD D = A + B
0011 SUB D = A - B
0100 MULT D = A * B
0110 EXP D = A POW B
0111 SQ D = A * A (ignores B)
1011 ACC D = A + 1
1100 DEC D = A - 1
1110 M16 D = A * 16
1111 M10 D = A * 10

Input

  • 18 switches -- left-most will be opcode (4), followed by A and B (3 each). Right-most 8-bits will be numerical input, if applicable.
  • Execute button -- this will do whatever operation is specified by the OpCode.

Output

The 7-segment display shows the contents of the first register (where the result of the operation will be stored). This value is also displayed in 8 green LEDs. The contents of different registers may be viewed by simply changing the value of A. (One does not need to push the execute button to change the display.)

Port Map

WRITE_ENABLE (the button/clock) is wired to PIN_G26.

Difficulties encountered

We encountered some difficulties with certain VHDL functions not being operable in Quartus. These were division and XOR. XOR was particularly important, and was going to be used for an XOR instruction, Invert instruction (XOR 0xFF), and the shifting instructions.

Some time was also wasted due to a file getting corrupted on a computer, some work was lost.

Conclusion

In conclusion, a working calculator was produced, but it was lacking some initial features due to difficulties programming. To simplify design, a tightly integrated design was used.

Proj1.vhd

library ieee;
use ieee.std_logic_1164.all;
use ieee.std_logic_arith.all;
use ieee.std_logic_unsigned.all;

entity proj1 is
  port(
    OPCODE : in std_logic_vector (3 downto 0);
    A : in unsigned (2 downto 0);
    B : in unsigned (2 downto 0);
    INPINS : in unsigned ( 7 downto 0 );
    OUTPINS : out unsigned ( 7 downto 0 ) ;
    SEVEN_SEG_L : out unsigned (6 downto 0 );
    SEVEN_SEG_H : out unsigned (6 downto 0 );
    WRITE_ENABLE : in std_logic
  );
end entity;

architecture behav of proj1 is
  type ram_type is array (0 to 7) of unsigned (7 downto 0);
  signal tmp_ram: ram_type;
begin
  process (WRITE_ENABLE)
  begin
    OUTPINS <= tmp_ram(conv_integer(A));
    if(WRITE_ENABLE'event AND WRITE_ENABLE = '1') then
      if( OPCODE = "0000" ) THEN
        tmp_ram(conv_integer(A)) <= tmp_ram(conv_integer(B));
      elsif( OPCODE = "0001" ) THEN
        tmp_ram(conv_integer(A)) <= INPINS;
      elsif( OPCODE = "0010" ) THEN
        tmp_ram(conv_integer(A)) <=
          tmp_ram(conv_integer(A)) + tmp_ram(conv_integer(B));
      elsif( OPCODE = "0011" ) THEN
        tmp_ram(conv_integer(A)) <=
          tmp_ram(conv_integer(A)) - tmp_ram(conv_integer(B));
      elsif( OPCODE = "0100" ) THEN
        tmp_ram(conv_integer(A)) <=
          tmp_ram(conv_integer(A)) * tmp_ram(conv_integer(B));
      elsif( OPCODE = "0110" ) THEN --POW
        if( tmp_ram( conv_integer(B) ) = 1) then
          tmp_ram(conv_integer(A)) <=
            tmp_ram(conv_integer(A));
        elsif( tmp_ram( conv_integer(B) ) = 2) then
          tmp_ram(conv_integer(A)) <=
            tmp_ram(conv_integer(A)) * tmp_ram(conv_integer(A));
        elsif( tmp_ram( conv_integer(B) ) = 3) then
          tmp_ram(conv_integer(A)) <=
            (tmp_ram(conv_integer(A)) * tmp_ram(conv_integer(A))
             * tmp_ram(conv_integer(A)));
        elsif( tmp_ram( conv_integer(B) ) = 4) then
          tmp_ram(conv_integer(A)) <=
            (tmp_ram(conv_integer(A)) * tmp_ram(conv_integer(A))
             * tmp_ram(conv_integer(A)) * tmp_ram(conv_integer(A)));
        elsif( tmp_ram( conv_integer(B) ) = 5) then
          tmp_ram(conv_integer(A)) <=
            (tmp_ram(conv_integer(A)) * tmp_ram(conv_integer(A))
             * tmp_ram(conv_integer(A)) * tmp_ram(conv_integer(A))
             * tmp_ram(conv_integer(A)));
        elsif( tmp_ram( conv_integer(B) ) = 6) then
          tmp_ram(conv_integer(A)) <=
            (tmp_ram(conv_integer(A)) * tmp_ram(conv_integer(A))
             * tmp_ram(conv_integer(A)) * tmp_ram(conv_integer(A))
             * tmp_ram(conv_integer(A)) * tmp_ram(conv_integer(A)));
        end if;
      elsif( OPCODE = "0111" ) THEN --SQ
        tmp_ram(conv_integer(A)) <=
          tmp_ram(conv_integer(A)) * tmp_ram(conv_integer(A));
      elsif( OPCODE = "1010" ) THEN --SL
        tmp_ram(conv_integer(A)) <= tmp_ram(conv_integer(A)) (6 downto 0) + 2;
      elsif( OPCODE = "1011" ) THEN --ACC
        tmp_ram(conv_integer(A)) <= tmp_ram(conv_integer(A)) + 1;
      elsif( OPCODE = "1100" ) THEN --DEC
        tmp_ram(conv_integer(A)) <= tmp_ram(conv_integer(A)) - 1;
      elsif( OPCODE = "1110" ) THEN --M16
        tmp_ram(conv_integer(A)) <=
          tmp_ram(conv_integer(A)) * conv_unsigned(16, 7);
      elsif( OPCODE = "1111" ) THEN --MTEN
        tmp_ram(conv_integer(A)) <=
          tmp_ram(conv_integer(A)) * conv_unsigned(10, 7);
      end if;
    end if;
    case tmp_ram(conv_integer(A)) (3 downto 0) is
      WHEN "0000" => SEVEN_SEG_L <= "1000000";
      WHEN "0001" => SEVEN_SEG_L <= "1111001";
      WHEN "0010" => SEVEN_SEG_L <= "0100100";
      WHEN "0011" => SEVEN_SEG_L <= "0110000";
      WHEN "0100" => SEVEN_SEG_L <= "0011001";
      WHEN "0101" => SEVEN_SEG_L <= "0010010";
      WHEN "0110" => SEVEN_SEG_L <= "0000010";
      WHEN "0111" => SEVEN_SEG_L <= "1111000";
      WHEN "1000" => SEVEN_SEG_L <= "0000000";
      WHEN "1001" => SEVEN_SEG_L <= "0010000";
      WHEN "1010" => SEVEN_SEG_L <= "0001000";
      WHEN "1011" => SEVEN_SEG_L <= "0000011";
      WHEN "1100" => SEVEN_SEG_L <= "1000110";
      WHEN "1101" => SEVEN_SEG_L <= "0100001";
      WHEN "1110" => SEVEN_SEG_L <= "0000110";
      WHEN "1111" => SEVEN_SEG_L <= "0001110";
      WHEN OTHERS => SEVEN_SEG_L <= "0111111";
    end case;
    case tmp_ram(conv_integer(A)) (7 downto 4) is
      WHEN "0000" => SEVEN_SEG_H <= "1000000";
      WHEN "0001" => SEVEN_SEG_H <= "1111001";
      WHEN "0010" => SEVEN_SEG_H <= "0100100";
      WHEN "0011" => SEVEN_SEG_H <= "0110000";
      WHEN "0100" => SEVEN_SEG_H <= "0011001";
      WHEN "0101" => SEVEN_SEG_H <= "0010010";
      WHEN "0110" => SEVEN_SEG_H <= "0000010";
      WHEN "0111" => SEVEN_SEG_H <= "1111000";
      WHEN "1000" => SEVEN_SEG_H <= "0000000";
      WHEN "1001" => SEVEN_SEG_H <= "0010000";
      WHEN "1010" => SEVEN_SEG_H <= "0001000";
      WHEN "1011" => SEVEN_SEG_H <= "0000011";
      WHEN "1100" => SEVEN_SEG_H <= "1000110";
      WHEN "1101" => SEVEN_SEG_H <= "0100001";
      WHEN "1110" => SEVEN_SEG_H <= "0000110";
      WHEN "1111" => SEVEN_SEG_H <= "0001110";
      WHEN OTHERS => SEVEN_SEG_H <= "0111111";
    end case;
  end process;
end architecture;
